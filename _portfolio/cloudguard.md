**Objective:** To build a production-ready system that automates the identification and quantification of security risks, bridging the gap between technical vulnerabilities and economic impact.

### The "Econ" Hook: Risk-to-Value Modeling
Unlike traditional scanners, CloudGuard uses a custom risk-scoring algorithm to estimate the **Potential Loss (Cost of Inaction)** vs. the **Remediation Cost (Cost of Security)**.

### Key Features
*   **Automated Governance:** Scans IAM and storage configurations using Python.
*   **Risk Scoring:** Assigns a "Risk Dollar Value" to each finding based on historical data breach costs.
*   **Architecture:** Integrated with Terraform for automated remediation and GitHub Actions for continuous auditing.

### Tech Stack
*   Python (3.11+), Azure Identity, Terraform, GitHub Actions.
